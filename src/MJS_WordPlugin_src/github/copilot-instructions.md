# Copilot リファクタリング指示書

このプロジェクトでは、C# 7.3 および .NET Framework 4.8 を使用しています。  
リファクタリングやコード生成時は、以下のガイドラインに従ってください。

## コーディング規約

- 既存のコーディングスタイル（インデント、命名規則、コメント形式）を尊重してください。
- 例外処理は必要最小限とし、catchブロックは具体的な例外型を指定してください。
- 可能な限り早期リターンを用いてネストを浅く保ってください。
- マジックナンバーやハードコーディングされた文字列は定数化してください。
- 冗長なコードや重複処理はメソッド化して共通化してください。
- 100行以上のコードを変更しないこと。

## ファイル・パス操作

- パス結合には `Path.Combine` を使用し、手動での文字列連結は避けてください。
- ファイル存在チェックや例外処理は適切に行ってください。

## 正規表現

- 正規表現パターンは定数として管理し、コメントで用途を明記してください。

## その他
- 自動生成コードには `[GeneratedCode]` 属性を付与してください。
- テストコードは `Tests` フォルダに配置してください。
