# GitHub Copilot リファクタリングガイド

このプロジェクトでGitHub Copilotを段階的にリファクタリングする際の指示・ガイドラインをまとめています。

## 基本方針

- **C# 7.3** および **.NET Framework 4.8** の仕様に準拠してください。
- 既存のコーディングスタイル（命名規則、インデント、コメント形式など）を尊重してください。
- 既存の機能を損なわないよう、動作確認・テストを必ず実施してください。

## リファクタリングの指針

- **メソッドの分割・整理**  
  長大なメソッドや複雑なロジックは、責務ごとに分割し、可読性・保守性を高めてください。
- **命名の明確化**  
  変数名・メソッド名・クラス名は、役割が明確に伝わるようにしてください。
- **重複コードの排除**  
  同じ処理が複数箇所に存在する場合は、共通メソッド化を検討してください。
- 重要な処理や複雑なロジックには、簡潔なコメントを付与してください。
- 100行以上のコードを変更しないでください。
- 自動生成コードには `[GeneratedCode]` 属性を付与してください。
- テストコードは `Tests` フォルダに配置してください。
- 動作が変わらないように慎重に変更してください。
